How Auth Context Works

How does login work in your frontend?
When user logs in, the backend returns a JWT token. We store this token in localStorage. Then our AuthContext — which wraps the entire app — makes the user object available everywhere using React's Context API. Every API call automatically attaches this token in the Authorization: Bearer <token> header using an Axios interceptor. When the token expires (after 24 hours), the interceptor catches the 401 error, clears localStorage, and shows "Session expired".

2.2 Backend (FastAPI + Python) — The brain

What is FastAPI?

FastAPI is a modern Python web framework for building APIs. It's very fast, has automatic data validation (using Pydantic), and generates API documentation automatically.

Explain the backend structure
The backend follows a layered architecture:
Layer 1: Routers (8 files) — handle HTTP requests, validate input, return responses. Like a receptionist.
Layer 2: Services (10 files) — contain business logic. Like the actual workers.
Layer 3: Database (1 file) — connects to MongoDB. Like the storage room.
Layer 4: Models (2 files) — define data shapes. Like the forms/templates.

For example: when user clicks "Optimize" → resume_router.py receives the request → calls openai_service.py (which calls Groq AI) → calls scoring_service.py (which calculates score) → returns everything to the user.

How main.py Works

Explain your main.py file
main.py is the entry point of the backend. Here's what happens when the server starts:
1. load_dotenv() — loads environment variables from .env file (API keys, database URL)
2. lifespan() — startup function that connects to MongoDB and creates the default admin account
3. FastAPI() — creates the app instance
4. CORSMiddleware — allows the frontend (on different domain) to talk to the backend
5. security_headers middleware — adds security headers to every response
6. include_router() — registers all 8 route modules with their URL prefixes
7. Rate limiter — prevents abuse (e.g., max 5 login attempts per minute)

How AI Integration Works

How does AI optimization work?
We use the Groq API with the Llama 3.3 70B model. Here's the flow:
1. User's resume data (JSON) is sent to our backend
2. openai_service.py builds a prompt: "Here's a resume and a job description. Optimize the resume, keep it truthful, improve bullets with metrics."
3. We send this prompt to Groq API using groq_client.py
4. Groq runs Llama 3.3 70B and returns optimized JSON
5. We parse the JSON, validate it with Pydantic, calculate ATS score
6. Return everything to the frontend

The key technique is asyncio.to_thread() — since Groq SDK is synchronous, we run it in a separate thread so the FastAPI server isn't blocked.

How ATS Scoring Works

How do you calculate ATS score?
The ATS score is calculated in scoring_service.py using 4 criteria:
1. Keyword Match (30%) — How many words from the job description appear in the resume? We tokenize both texts and calculate intersection.
2. Skills Alignment (25%) — How many of the user's listed skills match the job description keywords?
3. Experience Relevance (25%) — Do the work experience bullet points contain relevant keywords?
4. Formatting Compliance (20%) — Does the resume have contact info? 3+ skills? Work experience? Action verbs like "developed", "implemented"? 100+ words?

Formula: overall = keyword*0.30 + skills*0.25 + experience*0.25 + formatting*0.20

2.3 Database (MongoDB) — Where data lives

Explain your database design
I use MongoDB Atlas (cloud NoSQL database) with 3 collections:

1. users — stores username, email, hashed_password (bcrypt), is_admin, security_question, security_answer. Has unique indexes on email and username.

2. saved_resumes — stores user_id, title, resume_data (the full resume JSON), optimized_data, ats_score, template choice, share_token (for public sharing). Indexed by user_id.

3. resume_uploads — stores uploaded file analysis: user_id, filename, resume_text (extracted), ats_score, analysis results, predicted_jobs. Indexed by user_id.

I chose MongoDB over SQL because resume data is deeply nested — a resume has an array of experiences, each with an array of bullets. In SQL this would need 8+ tables with JOINs. In MongoDB, it's stored naturally as a single document.

2.4 Authentication — How login/security works

Explain your authentication system
I implemented JWT (JSON Web Token) authentication:
1. Registration: User sends username, email, password → password is hashed with bcrypt (one-way hash with salt) → stored in MongoDB → JWT token returned
2. Login: User sends email + password → we find user by email → bcrypt.checkpw() verifies password → generate JWT token (valid 24 hours)
3. Protected Routes: Token is sent in Authorization: Bearer <token> header → backend decodes it using jwt.decode() → extracts user_id → fetches user from DB
4. Password Recovery: Uses security question + answer (answer is also bcrypt-hashed)
5. Admin: Users with is_admin=True can access admin endpoints. Default admin created on startup.

🟡 Category 2: Technical Deep Dive

What is JWT? How does it work?
JWT (JSON Web Token) is a compact, URL-safe token for authentication. It has 3 parts separated by dots: Header (algorithm), Payload (user_id, username, expiry), Signature (hash of header+payload using our secret key). The server signs the token on login. On each request, we decode it to verify identity. We use HS256 algorithm and 24-hour expiry.
What is bcrypt? Why not store passwords directly?
Storing plain passwords is extremely dangerous — if the database is hacked, all passwords are exposed. bcrypt is a one-way hashing algorithm that converts "MyPassword123" to "$2b$12$LJ3m..." — you CANNOT reverse it. Each hash has a random salt, so even two users with the same password get different hashes. To verify login, bcrypt compares the hash of the entered password with the stored hash.
What is CORS? Why do you need it?
CORS (Cross-Origin Resource Sharing) is a security feature in browsers. By default, a website on domain A cannot make API calls to domain B. Our frontend is on vercel.app and backend is on render.com — different domains. CORS middleware tells the browser: "Yes, requests from this specific frontend URL are allowed." Without it, all API calls would be blocked.
What is a REST API?
REST (Representational State Transfer) is an architecture for web APIs. It uses HTTP methods: GET (read data), POST (create/submit), PUT (update), DELETE (remove). Each URL represents a resource. For example: POST /api/auth/login = login, GET /api/saved/list = get my resumes, DELETE /api/saved/123 = delete resume with id 123.
What is async/await? Why do you use it?
async/await lets Python do other work while waiting for slow operations. Example: When User A calls the AI (takes 8 seconds), instead of blocking the entire server, async allows User B's request to be handled during that wait. Without async, 100 concurrent users would have to wait in line. With async, the server handles them all simultaneously.
What is asyncio.to_thread()? Why do you use it?
The Groq SDK is synchronous — it blocks until the AI responds. But FastAPI's event loop is single-threaded. If we call Groq directly, the entire server freezes for 5-10 seconds. asyncio.to_thread() solves this by running the Groq call in a separate thread, so the main event loop stays free to handle other requests.
What is Pydantic?
Pydantic is a Python library for data validation. We define a model like class RegisterRequest(BaseModel): email: EmailStr; password: str = Field(min_length=6). When a user sends a request, Pydantic automatically checks: Is email valid? Is password at least 6 characters? If not, it returns a clear error — no manual if/else needed.
What is rate limiting?
Rate limiting restricts how many requests a user can make in a time window. We use SlowAPI library. Examples: login is limited to 10 requests/minute, register to 5/minute, AI optimization to 10/minute. This prevents brute-force attacks (trying thousands of passwords) and protects against abuse of our AI API quota.
What are environment variables? Why use them?
Environment variables store secret configuration like API keys, database passwords, and JWT secrets. They're set on the server (not in code). Reason: if code is on GitHub (public), anyone could see the secrets. By using .env file locally and Render/Vercel environment settings in production, secrets are never in the codebase.
What is middleware?
Middleware is code that runs before or after every request. Like a security checkpoint at a building entrance. In our app: (1) CORS middleware — checks if the frontend's domain is allowed, (2) Security headers middleware — adds X-Frame-Options, X-Content-Type-Options to prevent attacks, (3) Rate limiter — counts requests per IP and blocks if exceeded.
Explain the Axios interceptor pattern
Interceptors are functions that run on every HTTP request/response. Request interceptor: Before every API call, we automatically add the JWT token to the Authorization header — so we don't repeat this code in every function. Response interceptor: If any call returns 401 (unauthorized), we auto-logout the user and show "Session expired". If 429 (rate limited), we show "Too many requests".

🟢 Category 3: React / Frontend Specific

What is useState?
useState is a React Hook that lets a component "remember" data. Example: const [step, setStep] = useState(0) — step holds the current value (starts at 0), setStep is the function to change it. When setStep(1) is called, React re-renders the component with the new value. We use it for form inputs, loading states, dark mode toggle, etc.
What is useEffect?
useEffect runs side effects — code that happens outside of rendering. It runs after the component renders. Example: useEffect(() => { getProfile() }, []) — the empty [] means "run once on mount". We use it for: fetching user profile on login, saving dark mode preference to localStorage, attaching event listeners.
What is useContext?
useContext accesses shared data without passing it through every component. Instead of passing user from App → Navbar → Button (prop drilling), we create a Context that wraps the app, and any component can directly access user via useAuth(). We use it for: auth state (user, login, logout functions).
What is React Router?
React Router handles client-side navigation. When user clicks "Builder", the URL changes to /builder and the BuilderPage component loads — without a full page reload. This makes the app feel instant. We define routes like: <Route path="/builder" element={<BuilderPage />} />.
What is code splitting / lazy loading?
Instead of loading all 11 pages when the app starts, we use lazy() to load each page only when the user visits it. Example: const AdminPage = lazy(() => import('./pages/AdminPage')). This makes the initial load much faster — the user only downloads what they need. Suspense shows a loading spinner while the page downloads.
What is a Protected Route?
A wrapper component that checks if the user is logged in before showing a page. If not logged in, it redirects to the landing page. For admin pages, it also checks user.is_admin. Example: <ProtectedRoute adminOnly><AdminPage /></ProtectedRoute> — only admin users can access this route.
What is Tailwind CSS?
Tailwind is a utility-first CSS framework. Instead of writing CSS classes like .btn-primary { background: blue; padding: 10px; }, you write utility classes directly: className="bg-blue-500 p-4 rounded-lg". Benefits: no context-switching between HTML and CSS files, consistent design system, responsive design with md: and lg: prefixes, dark mode with dark: prefix.

🔵 Category 4: Advanced / If Maam Digs Deep

What is the difference between sync and async database?
We use two database connections: pymongo (synchronous) for simple operations like user login (which is fast), and motor (asynchronous) for database initialization and index creation. The sync connection is used inside asyncio.to_thread() when needed. This dual approach gives us simplicity for most operations and non-blocking behavior for heavy ones.
How do you prevent SQL injection? (Or: How do you prevent NoSQL injection?)
Since we use MongoDB (not SQL), SQL injection doesn't apply. But NoSQL injection is possible. We prevent it by: (1) Pydantic validation — only allows expected data types and formats. (2) Parameterized queries — PyMongo uses BSON, not string concatenation. (3) Input sanitization — email is validated with EmailStr, filenames are stripped of special characters.
What security measures did you implement?
10 measures: (1) bcrypt password hashing, (2) JWT authentication, (3) Rate limiting on all sensitive endpoints, (4) CORS origin restriction, (5) Security headers (X-Frame-Options, HSTS), (6) Email enumeration prevention, (7) Filename sanitization in PDF downloads, (8) Admin password must be set via environment in production, (9) Hashed security question answers, (10) Input validation with Pydantic.
What is a Progressive Web App (PWA)?
A PWA is a website that can be installed on your phone/desktop like a native app. Our app has: (1) manifest.json — defines app name, icon, theme color, (2) service-worker.js — caches static assets for offline use, (3) Cache versioning — when we deploy updates, the old cache is automatically cleared.
What is the difference between Llama model and GPT?
Both are Large Language Models (LLMs). GPT is made by OpenAI (closed source, expensive). Llama is made by Meta (open source). We use Llama 3.3 70B — "70B" means 70 billion parameters (weights the model learned during training). We access it via Groq API — Groq has custom hardware that runs inference very fast (faster than OpenAI for Llama models).
What if Groq API is down? How do you handle errors?
We have error handling at multiple levels: (1) try/except in router — catches AI errors and returns HTTP 502 with message, (2) 60-second timeout in frontend — if AI takes too long, shows error, (3) toast.error() shows user-friendly message, (4) The heuristic scoring (no AI) always works as fallback — users can use quick-score without AI. (5) ErrorBoundary component catches any React crash and shows recovery UI.
How does the resume PDF generation work?
We use xhtml2pdf library. The flow: (1) Resume data → Jinja2 HTML template → HTML string, (2) xhtml2pdf converts HTML to PDF bytes, (3) FastAPI returns PDF as binary with Content-Type: application/pdf and Content-Disposition: attachment header. For DOCX, we use python-docx library to build the document programmatically (paragraph by paragraph).
What is the lifespan function in FastAPI?
lifespan is FastAPI's way to run code on startup and shutdown. On startup: we connect to MongoDB, create database indexes, and seed the default admin account. On shutdown: we close all database connections cleanly. It uses Python's asynccontextmanager — code before yield runs on startup, code after runs on shutdown.
What are database indexes? Why did you create them?
Indexes make queries fast — like a book's index. Without indexes, MongoDB scans every document to find a match (slow with 10,000+ users). We create: (1) Unique index on email — prevents duplicate registrations and makes login lookup instant, (2) Unique index on username, (3) Index on user_id in saved_resumes — fast lookup of a user's resumes, (4) Sparse index on share_token — fast public resume lookup.

Part 7: Key Terms Dictionary

TermSimple Explanation
APIA way for two programs to talk to each other. Our frontend sends data to backend via API.
RESTA style of API using HTTP methods (GET, POST, PUT, DELETE) to manage resources.
JWTA signed token sent with every request to prove "I am this user". Like a digital ID card.
bcryptOne-way password hashing. Converts password to irreversible hash for safe storage.
CORSBrowser security that blocks cross-domain API calls unless the server explicitly allows it.
MiddlewareCode that runs before/after every request — like a security guard at every door.
Hook (React)Functions like useState, useEffect that add capabilities to React components.
ComponentA reusable piece of UI (button, form, card). React app = tree of components.
PropsData passed from parent component to child component. Like function parameters.
StateData that can change and triggers re-render. Managed with useState.
ContextShared data accessible by any component without prop drilling through every level.
PydanticPython library for automatic data validation using type hints.
MongoDBNoSQL database that stores data as JSON-like documents (not tables with rows).
CollectionMongoDB's equivalent of a SQL table. We have: users, saved_resumes, resume_uploads.
IndexDatabase structure that speeds up queries. Like a book's index for fast lookup.
async/awaitNon-blocking code execution. Server handles other requests while waiting for slow operations.
Virtual DOMReact's optimization — compares old and new UI, updates only what changed.
Lazy LoadingLoading code only when needed. Pages load on navigation, not all at once.
Rate LimitingLimiting requests per time window (e.g., 10/minute) to prevent abuse.
Environment VariableServer-side config (API keys, secrets) stored outside code for security.
LLMLarge Language Model — AI trained on massive text data. Llama 3.3 is one.
Prompt EngineeringCrafting the right instructions for AI to get desired output format and quality.
TokenizationBreaking text into individual words/pieces for comparison and analysis.
PWAProgressive Web App — website installable like a phone app, with offline support.
Service WorkerScript running in background that caches files for offline use and push notifications.
ViteFast build tool for React (replaces older tools like Webpack). Instant hot reload.
Tailwind CSSCSS framework using utility classes (bg-blue-500, p-4) instead of writing custom CSS.
Framer MotionReact animation library for smooth page transitions and element animations.
AxiosHTTP client library. Better than fetch() — has interceptors, automatic JSON, timeouts.
